DyTox Strategy

Focuses on Class Expansion. It assumes the visual domain is constant but the vocabulary grows. It solves this by adding specialized "query tokens" to a shared Transformer backbone.

ABDF Strategy

Focuses on Domain Expansion. It assumes the vocabulary (Lanes) is constant but the environment changes. It solves this by creating physical branches and routing inputs based on appearance.
